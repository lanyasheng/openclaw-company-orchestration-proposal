## Summary

Successfully wired the unified handoff schema into the actual task registration flow. The `registration_handoff` generated by trading/channel roundtables is now actually registered to the task registry via `register_from_handoff()`.

## Changes

  - `task_id`

## Design Principles Followed

1. **Backward Compatible**: Existing registration paths preserved; new path is additive
2. **No Duplication**: Uses `register_from_handoff()` helper, doesn't rebuild registration payload
3. **Minimal Wiring**: Only added the actual registration call + return value metadata
4. **Safety Gates**: `registration_status` correctly derived from `allow_auto_dispatch` in safety_gates

## Handoff Flow Now Complete

```
DispatchPlan (from roundtable)
    ↓ to_planning_handoff()
PlanningHandoff
    ↓ build_registration_handoff()
RegistrationHandoff
    ↓ register_from_handoff()  ← P0-2 Batch 3
TaskRegistrationRecord (in registry)
    ↓ query via get_registration()
Consumer (operator/main/dispatcher)
```
