# Validation 文档区

这里存放的是：
- P0 审计结论
- readiness review
- bridge simulator
- human-gate / callback / task registry 契约
- 各类 repo-local POC 证明材料
- 历史设计稿与阶段性评审快照

## 先看什么

这些文档**提供证据**，但很多并不是“今天默认口径”的最佳入口。

在进入本目录前，建议先看：
1. `../executive-summary.md`
2. `../CURRENT_TRUTH.md`
3. `../openclaw-company-orchestration-proposal.md`
4. `../validation-status.md`
5. `../roadmap.md`

## 定位

- 主线文档回答：**总方案是什么 / 当前 live 真值是什么 / 边界在哪里**
- validation 文档回答：**某个判断的证据是什么**
- 历史设计稿与评审文档回答：**当时为什么这样判断，以及后来哪些点被更新了**

## 当前治理口径

- 本目录内若看到 `historical / outdated / superseded` 标注，请按标注处理
- 没有标注也不代表它就是今天的默认入口；默认仍以 `../CURRENT_TRUTH.md` 与 `../validation-status.md` 为准
- 不允许再让局部 POC、旧计划文档或单次审查报告覆盖仓库总方案口径

## 一句话规则

**先看主线，再用 validation 补证据；不要反过来。**
